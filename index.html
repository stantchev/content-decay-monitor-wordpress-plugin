WP-Cron automation

Daily background scans â€” no manual work

A transient-based offset lets each daily cron fire advance the scan window safely across large sites.

STEP 01

Plugin Activates

Registers cdm_daily_scan WP-Cron event on activation.

STEP 02

Cron Fires

Once per day, retrieves the stored transient offset (defaults to 0 for a fresh cycle).

STEP 03

Batch Scanned

Scans up to 20 posts (configurable). Updates _cdm_score meta on each.

STEP 04

Offset Advances

Saves new offset to transient. Next fire resumes from there, resets when all posts are done.

Quick answers about Content Decay Monitor.

Does scanning affect site performance?

No. Scans run manually via AJAX or via the background WP-Cron job. The scanner uses fields=ids queries â€” no full post objects loaded unnecessarily. Results are cached in post meta, so no computation happens on the front end or on page load.

Yes â€” use the cdm_post_types filter. See the Customization section above for an example.

What happens if I have thousands of posts?

The daily WP-Cron job scans in batches (default 20) and stores its offset in a transient. Each daily fire advances the window until all posts are covered, then the cycle resets â€” no timeouts, no memory pressure.
